# 架构说明

## 目标定位

当前版本是“科研平台化原型”，强调以下目标：

- 可运行：本地和 Docker Compose 都能快速启动
- 可演示：覆盖登录、数据录入、质量检查、推荐、报告、模型中心和因果结果页
- 可复现：保存训练参数、随机种子、特征键、复现状态和模型产物
- 可扩展：规则引擎、因果估计器和页面工作台都预留替换空间

## 技术栈

- 后端：FastAPI + SQLAlchemy + Pydantic + SQLite
- 前端：Vue3 + Vite + TypeScript + Element Plus + Pinia + Vue Router + ECharts
- 测试：Pytest + Vitest
- 持续集成：GitHub Actions
- 编排：Docker Compose

## 分层设计

- 前端负责登录演示、管理台壳、数据工作台、质量联动、模型版本对比与结果展示。
- 后端提供认证、患者管理、导入、质量检查、规则推荐、报告生成与因果训练接口。
- 数据层包含患者、量表、睡眠、光干预、随访、推荐结果、模型版本和审计日志。
- `modeling` 模块负责因果数据读取、特征筛选、训练/验证拆分和估计器适配。

## 关键能力

- 数据质量接口返回结构化 `summary / blocking_issues / warning_issues / suggested_fixes / affected_patient_ids`，可被质量页、数据中心和模型中心复用。
- 模型中心记录随机种子、覆盖率阈值、选中特征键、训练/验证样本量、估计后端和复现状态，支持参数复用和按既有配置重训。
- 推荐中心明确区分“规则/评分版推荐”与“因果获益评估”，避免科研演示中混淆两类结论。
- 前端使用路由级懒加载和 vendor 拆包，显著改善单包体积。

## 当前边界

- 仅供科研辅助，不替代临床诊断与治疗。
- 不做真实 HIS 对接。
- 不做设备实时接入。
- 不做移动端。
- 不引入正式生产级权限体系和临床级模型部署流程。
